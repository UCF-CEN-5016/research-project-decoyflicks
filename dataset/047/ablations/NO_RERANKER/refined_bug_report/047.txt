### TITLE
MMS ASR Inference Output Log Order is Incorrect with Multiple Audio Inputs

### SYMPTOMS
• The output log generated during the MMS ASR inference does not match the order of the input audio files.  
• The expected text output is not aligned with the corresponding audio file paths.  
• Example of disorder in output:
  ```
  Expected order:
  /Dataset/speech/english/test/libri_test_other/wav/wav/1/wav/1/1089-134686-0000.wav → Output: "at most by an alms given to a beggar..."
  /Dataset/speech/english/test/libri_test_other/wav/wav/1/wav/1/1089-134686-0001.wav → Output: "the dull light fell more faintly..."
  ...
  
  Actual order:
  Input: /Dataset/speech/english/test/libri_test_other/wav/wav/1/wav/1/1089-134686-0000.wav
  Output: at most by an alms given to a beggar whose blessing he fled from he might hope wearily to win for himself some measure of actual grace
  ...
  ```

### EXPECTED BEHAVIOR
The output log should maintain the same order as the input audio files, ensuring that each audio file corresponds to its expected text output in the correct sequence.

### REPRODUCTION STEPS
1. Run the following command:
   ```bash
   python -u examples/mms/asr/infer/mms_infer.py \
       --model `pwd`/../fairseq_resource/mms1b_all.pt \
       --lang "eng" \
       --audio audio1.wav audio2.wav audio3.wav ... audio10.wav
   ```
2. Observe the output log generated by the inference process.
3. Compare the order of the input audio files with the corresponding output text to identify discrepancies.
