### TITLE
Incorrect Output Order in MMS ASR Inference with Multiple Audio Files

### SYMPTOMS
• The output log from the MMS ASR inference does not match the order of the input audio files.  
• The inference results for audio files are printed in a disordered manner.  
• The expected output log should match the order of the input audio files.  

**Example of the disorder:**
- Input order:
  ```
  1089-134686-0000
  1089-134686-0001
  1089-134686-0002
  ...
  1089-134686-0009
  ```
- Output order:
  ```
  Input: /Dataset/speech/english/test/libri_test_other/wav/wav/1/wav/1/1089-134686-0000.wav
  Output: at most by an alms given to a beggar whose blessing he fled from he might hope wearily to win for himself some measure of actual grace
  ...
  Input: /Dataset/speech/english/test/libri_test_other/wav/wav/1/wav/1/1089-134686-0009.wav
  Output: allo berti any good in your mind
  ```

### EXPECTED BEHAVIOR
The output log should reflect the same order as the input audio files, ensuring that each audio file's inference result corresponds correctly to its respective input.

### REPRODUCTION STEPS
1. Run the following command:
   ```bash
   python -u examples/mms/asr/infer/mms_infer.py \
       --model `pwd`/../fairseq_resource/mms1b_all.pt \
       --lang "eng" \
       --audio audio1.wav audio2.wav audio3.wav ... audio10.wav
   ```
2. Observe the output log generated by the inference process.
3. Compare the order of the output text with the order of the input audio files.
