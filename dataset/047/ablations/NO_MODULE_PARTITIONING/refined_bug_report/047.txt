### TITLE
MMS ASR Inference Output Order is Incorrect

### SYMPTOMS
• The output log from the MMS ASR inference does not match the order of the input audio files.  
• The audio inference outputs are misaligned with their respective input files, causing confusion in the results.  
• The expected transcription order is not maintained, leading to a disorder in the output log.  
• Example of expected output order:  
  ```
  1089-134686-0000: "HE HOPED THERE WOULD BE STEW FOR DINNER..."
  1089-134686-0001: "STUFF IT INTO YOU HIS BELLY COUNSELLED HIM"
  ...
  ```
• Example of actual output log order:
  ```
  Input: /Dataset/speech/english/test/libri_test_other/wav/wav/1/wav/1/1089-134686-0000.wav
  Output: "at most by an alms given to a beggar..."
  ...
  ```

### EXPECTED BEHAVIOR
The output logs from the MMS ASR inference should maintain the same order as the input audio files, ensuring that each transcription corresponds correctly to its respective input file.

### REPRODUCTION STEPS
1. Run the following command to execute the ASR inference:
   ```bash
   python -u examples/mms/asr/infer/mms_infer.py \
           --model `pwd`/../fairseq_resource/mms1b_all.pt \
           --lang "eng" \
           --audio audio1.wav audio2.wav audio3.wav ... audio10.wav
   ```
2. Observe the output log generated by the inference process.
3. Compare the order of the input audio files with the output transcriptions to identify any discrepancies.
