TITLE  
NaN logits in decoder when cross-attention context is entirely padding

SYMPTOMS  
• Decoder outputs NaN logits when the context provided to the cross-attention block consists entirely of padding.  
• Example output: `out is all NaN`.  
• The context mask is entirely set to `False`, indicating no valid context tokens are provided.

EXPECTED BEHAVIOR  
When the context provided to the cross-attention block is all padding, the decoder should either produce valid logits (potentially with a default behavior) or handle the situation gracefully without resulting in NaN values.

REPRODUCTION STEPS  
1. Initialize the `TransformerWrapper` with a `Decoder` that includes cross-attention.  
2. Create a random input tensor `i` with shape `(2, 20)` using `torch.randint`.  
3. Create a context tensor `context` with shape `(2, 4, 1024)` using `torch.rand`.  
4. Create a context mask `context_mask` with shape `(2, 4)` filled with `False`.  
5. Call the decoder with the input tensor and the context (and mask) using `out = decoder(i, context=context, context_mask=context_mask)`.  
6. Observe the output, which should be NaN.  

By following these steps, the issue can be consistently reproduced, highlighting the need for resolution regarding the handling of padding in the cross-attention context.
