TITLE  
Validation loss consistently reporting zero in Instance/Semantic Segmentation with Model Garden

SYMPTOMS  
• Validation loss remains at `0.0` during evaluation for instance segmentation.  
• The same issue is observed in the official tutorials for both instance and semantic segmentation.  
• Example output showing validation loss:  
  ```
  eval | step:   1200 | steps/sec:    3.3 | eval time:   60.0 sec | output: 
  {'AP': 0.0814979,
   'AP50': 0.16509584,
   'AP75': 0.07145937,
   ...
   'mask_ARs': 0.0015765766,
   'steps_per_second': 3.3340747415649394,
   'validation_loss': 0.0}
  ```  
• The same behavior is noted in the Semantic Segmentation tutorial, where validation loss is also reported as zero.

EXPECTED BEHAVIOR  
The validation loss should reflect a meaningful value greater than zero during the evaluation phase, indicating that the model is learning and generalizing from the training data.

REPRODUCTION STEPS  
1. Open the notebook for Instance Segmentation with Model Garden (link: [Instance Segmentation with Model Garden 1](https://www.tensorflow.org/tfmodels/vision/instance_segmentation)).  
2. Run the training and evaluation sections of the notebook.  
3. Observe the output metrics during the evaluation phase, specifically looking for the `validation_loss` value.  
4. Repeat the steps in the official Semantic Segmentation tutorial (link: [Semantic Segmentation with Model Garden](https://www.tensorflow.org/tfmodels/vision/semantic_segmentation#train_and_evaluate)) and check for the same issue.
