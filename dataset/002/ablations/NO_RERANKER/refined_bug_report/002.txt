TITLE  
ModuleNotFoundError: No module named 'tensorflow.contrib' in TensorFlow 2.x

SYMPTOMS  
• Encountering a `ModuleNotFoundError` when running the training script.  
• Error message: `ModuleNotFoundError: No module named 'tensorflow.contrib'`  
• The issue arises specifically from the import statement `from tensorflow.contrib.quantize.python import graph_matcher` in the `exporter.py` file.  
• The user has attempted to resolve the issue by installing and importing `tf_slim`, but the error persists.

EXPECTED BEHAVIOR  
The script should execute without errors, successfully importing all necessary modules and training the model as intended.

REPRODUCTION STEPS  
1. Set up the environment with Debian 12 and Miniforge3.  
2. Install TensorFlow 2.x.  
3. Run the command: `python model_main_tf2.py --model_dir=models/centernet_mobilenetv2fp --pipeline_config_path=models/centernet_mobilenetv2fp/pipeline.config`.  
4. Observe the resulting error related to the `tensorflow.contrib` module.
