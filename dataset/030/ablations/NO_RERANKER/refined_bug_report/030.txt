**TITLE**  
MisconfigurationException during nnUNet training with PyTorch Lightning

**SYMPTOMS**  
• Encountered a `MisconfigurationException` when attempting to train the nnUNet model using BraTS21.ipynb and BraTS22.ipynb.  
• The error message indicates an issue with the learning rate scheduler:  
  ```plaintext
  raise MisconfigurationException(
  pytorch_lightning.utilities.exceptions.MisconfigurationException: The provided lr scheduler `CosineAnnealingWarmRestarts` doesn't follow PyTorch's LRScheduler API. You should override the `LightningModule.lr_scheduler_step` hook with your own logic if you are using a custom LR scheduler.
  ```  
• The training process starts with the message:  
  ```plaintext
  Provided checkpoint None is not a file. Starting training from scratch.
  ```  
• The training configuration shows that the model is set to use a GPU, but fails due to the misconfiguration.

**EXPECTED BEHAVIOR**  
The nnUNet model should train successfully without raising exceptions, utilizing the specified learning rate scheduler and completing the training process as intended.

**REPRODUCTION STEPS**  
1. Open BraTS21.ipynb or BraTS22.ipynb.  
2. Execute the cell responsible for training the nnUNet model.  
3. Run the following command:  
   ```plaintext
   !python ../main.py --brats --brats22_model --scheduler --learning_rate 0.0003 --epochs 10 --fold 0 --gpus 1 --task 11 --nfolds 10 --save_ckpt
   ```  
4. Observe the raised `MisconfigurationException` related to the learning rate scheduler.  

**ADDITIONAL NOTES**  
- Ensure that all required packages from `requirements.txt` are installed and that any additional dependencies are also met.  
- Verify that the learning rate scheduler is compatible with the PyTorch Lightning API or consider implementing the `lr_scheduler_step` hook if using a custom scheduler.
