**TITLE**  
RuntimeError: Loss value is NaN when training resnet_rs_imagenet model in TensorFlow 2.6.0

**SYMPTOMS**  
• A `RuntimeError` occurs during training, specifically stating:  
  `RuntimeError: The loss value is NaN after training loop and it happens 1 times.`  
• Loss values fluctuate and eventually become NaN after several training steps.  
• The training process fails to complete, preventing model evaluation and saving of results.

**EXPECTED BEHAVIOR**  
The training process should complete successfully without any errors, producing valid loss values throughout the training steps, allowing the model to be trained and evaluated.

   `docker pull nvcr.io/nvidia/tensorflow:21.12-tf2-py3`


   ```
   export PYTHONPATH=$(realpath ../../../):$PYTHONPATH  
   export TF_FORCE_GPU_ALLOW_GROWTH=true  
   ```

   `git checkout r2.6.0`
