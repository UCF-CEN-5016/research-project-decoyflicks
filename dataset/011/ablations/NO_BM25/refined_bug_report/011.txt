**TITLE**  
RuntimeError: The loss value is NaN during training of resnet_rs_imagenet on TensorFlow 2.6.0

**SYMPTOMS**  
• The model training process fails with a `RuntimeError: The loss value is NaN after training loop and it happens 1 times.`  
• Loss values printed during training include valid numbers followed by NaN values, indicating a failure in loss computation.  
• Example loss values observed before failure:  
  - {'loss': 7.83986902}  
  - {'loss': 167.242157}  
  - {'loss': nan} (and subsequent entries also show NaN).  

**EXPECTED BEHAVIOR**  
The model should train without errors, producing valid loss values throughout the training process, allowing for successful model evaluation and checkpointing.
