### TITLE
RuntimeError: The loss value is NaN during training of resnet_rs_imagenet in TF2 r2.6.0

### SYMPTOMS
• Encountering a `RuntimeError: The loss value is NaN after training loop and it happens 1 times.` during training.
• Loss values printed during training include valid numbers initially but eventually become NaN:
  - Example loss values: `{'loss': 7.83986902}`, `{'loss': 167.242157}`, `{'loss': nan}`.
• The training process fails after several steps due to the NaN loss value.

### EXPECTED BEHAVIOR
The model should train without errors, producing valid loss values throughout the training process, with no NaN values encountered.

1. Pull the Docker image: `nvcr.io/nvidia/tensorflow:21.12-tf2-py3`.
2. Install the required Python packages using pip:
   ```
   pip install gin-config sentencepiece seqeval pycocotools opencv-python sacrebleu jupyter-tensorboard tensorboard tensorboard-data-server tensorboard-plugin-wit tensorflow tensorflow-addons tensorflow-datasets tensorflow-estimator tensorflow-hub tensorflow-metadata tensorflow-model-optimization tensorflow-text
   ```
3. Check out the TensorFlow Model Garden repository at version r2.6.0:
   ```
   git checkout r2.6.0
   ```
4. Navigate to the model directory:
   ```
   cd official/vision/beta
   ```
5. Set the necessary environment variables:
   ```
   export PYTHONPATH=$(realpath ../../../):$PYTHONPATH
   ```


- The issue occurs in the TensorFlow Model Garden version r2.6.0 while using the resnet_rs_imagenet model.
