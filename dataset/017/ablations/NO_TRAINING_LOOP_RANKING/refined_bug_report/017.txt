TITLE  
Incompatibility of `object_detection_tutorial.ipynb` with TensorFlow 1.x

SYMPTOMS  
• The function `load_model` fails to load the model correctly, resulting in empty tensors.  
• The output shows `INFO:tensorflow:Saver not created because there are no variables in the graph to restore`.  
• The inference loop produces unexpected results, as indicated by the empty output image.  

EXPECTED BEHAVIOR  
When the software works correctly, the model should load successfully, and the inference loop should display the detected objects in the provided images.

REPRODUCTION STEPS  
1. Open the `object_detection_tutorial.ipynb` notebook located in the `colab_tutorials` folder.  
2. Attempt to run the `load_model` function with a valid model name for TensorFlow 1.x.  
3. Execute the inference loop using `show_inference(detection_model, image_path)` for a set of test images.  
4. Observe the console output and the results displayed in the notebook.
