TITLE  
Potential bug in `model.generate` when decoding shorter sequences

SYMPTOMS  
• The `model.generate` function fails to handle decoding sequences that are shorter than the encoding sequences.  
• Error message:  
  ```
  RuntimeError: The size of tensor a (258) must match the size of tensor b (230) at non-singleton dimension 3
  ```  
• The function call leads to a traceback indicating a mismatch in tensor sizes during the masked fill operation.

EXPECTED BEHAVIOR  
The `model.generate` function should successfully generate output sequences regardless of whether they are shorter or longer than the input encoding sequences.

REPRODUCTION STEPS  
1. Clone the repository from [toy_tasks](https://github.com/lucidrains/x-transformers/tree/main/examples/toy_tasks).  
2. Run the example code provided in the gist: [Reproducible example](https://gist.github.com/guillaumeguy/dd8993a622663db39aa1c54be9a875fd).  
3. Observe the error when calling `model.generate` with a shorter decoding sequence than the encoding sequence.
