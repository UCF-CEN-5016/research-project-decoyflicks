TITLE  
Potential bug in `model.generate` for shorter decoding sequences

SYMPTOMS  
• The `model.generate` function fails when attempting to decode sequences that are shorter than the encoding sequences.  
• Error message:  
```
RuntimeError: The size of tensor a (258) must match the size of tensor b (230) at non-singleton dimension 3
```

EXPECTED BEHAVIOR  
The `model.generate` function should successfully decode sequences regardless of their length compared to the encoding sequences, without raising any runtime errors.

REPRODUCTION STEPS  
1. Clone the repository from [toy_tasks](https://github.com/lucidrains/x-transformers/tree/main/examples/toy_tasks).  
2. Run the example script located at `/home/guillaume_guy/repos/pytorch-seq2seq/example.py`.  
3. Ensure that the `src` input and `start_tokens` are set up such that the decoding sequence length is shorter than the encoding sequence length.  
4. Execute the `model.generate` function with the parameters: `src`, `start_tokens`, `ENC_SEQ_LEN`, and `mask = src_mask`.  
5. Observe the error message generated during execution.
