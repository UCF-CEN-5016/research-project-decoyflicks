TITLE  
Script execution fails due to unexpected keyword argument in ParquetConfig

SYMPTOMS  
• The script `test_tune.sh` fails to execute properly.  
• Encountered error message: `ParquetConfig.__init__() got an unexpected keyword argument 'token'`.  
• The script attempts to run autotuning but reports: "No optimal DeepSpeed configuration found by autotuning."  
• SSH permission errors are noted during execution: "localhost: Permission denied, please try again."  

EXPECTED BEHAVIOR  
The script should successfully execute the autotuning process without errors, leading to the identification of an optimal DeepSpeed configuration for the model.

REPRODUCTION STEPS  
1. Clone the Hugging Face Transformers repository:  
   ```bash
   cd ~/  
   git clone https://github.com/huggingface/transformers.git  
   cd transformers  
   pip install .  
   ```
2. Create a new directory for testing and install DeepSpeed:  
   ```bash  
   mkdir ../test && cd ../test  
   pip install deepspeed # and any other necessary packages  
   ```
3. Create and modify the `test_tune.sh` script:  
   ```bash  
   touch test_tune.sh  
   chmod +x test_tune.sh  
   ```
4. Copy the contents of the original `test_tune.sh` file from [`autotuning/hf/gpt2-xl/test_tune.sh`](https://github.com/microsoft/DeepSpeedExamples/blob/ff9a0234cf22dd9af03c5c7aa8037fb9143adca6/training/autotuning/hf/gpt2-xl/test_tune.sh) into the new file.  
5. Change the values of `HF_PATH` to `../` and `NGPUS` to `6` in the script.  
6. Execute the script:  
   ```bash  
   ./test_tune.sh tune  
   ```  
7. Observe the output and errors during execution.
