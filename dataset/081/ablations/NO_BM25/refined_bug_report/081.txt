TITLE  
CrossViT Fails to Process Non-Three Channel Images

SYMPTOMS  
• CrossViT does not accept images with a channel count other than three, leading to runtime errors.  
• The following error message is observed:  
  ```
  RuntimeError: Given normalized_shape=[768], expected input with shape [*, 768], but got input of size[1, 256, 256]
  ```

EXPECTED BEHAVIOR  
The CrossViT model should be able to process images with varying channel counts (e.g., 1-channel, 4-channel) without throwing runtime errors.

REPRODUCTION STEPS  
1. Import the necessary libraries and modules:  
   ```py
   import torch
   from vit_pytorch.cross_vit import CrossViT
   ```
2. Initialize the CrossViT model with the desired parameters:  
   ```py
   v = CrossViT(
       image_size = 256,
       num_classes = 1000,
       depth = 4,
       sm_dim = 192,
       sm_patch_size = 16,
       sm_enc_depth = 2,
       sm_enc_heads = 8,
       sm_enc_mlp_dim = 2048,
       lg_dim = 384,
       lg_patch_size = 64,
       lg_enc_depth = 3,
       lg_enc_heads = 8,
       lg_enc_mlp_dim = 2048,
       cross_attn_depth = 2,
       cross_attn_heads = 8,
       dropout = 0.1,
       emb_dropout = 0.1
   )
   ```
3. Create an input tensor with a different number of channels (e.g., 1 channel):  
   ```py
   img = torch.randn(1, 1, 256, 256)  # Using 1 channel
   ```
4. Attempt to make a prediction using the model:  
   ```py
   pred = v(img)  # (1, 1000)
   ```  
5. Observe the runtime error that occurs during the prediction step.
