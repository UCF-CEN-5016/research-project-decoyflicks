### TITLE
CrossViT Fails to Process Images with Non-Three-Channel Formats

### SYMPTOMS
• The model does not accept images that do not have three channels (e.g., grayscale images with a single channel).
• The following error message is produced when attempting to process a single-channel image:
```
RuntimeError: Given normalized_shape=[768], expected input with shape [*, 768], but got input of size[1, 256, 256]
```
• The model appears to be designed exclusively for three-channel RGB images, leading to failures when provided with different channel configurations.

### EXPECTED BEHAVIOR
The CrossViT model should be able to accept images with varying numbers of channels (e.g., 1-channel, 3-channel, or more) and process them correctly without throwing errors.

### REPRODUCTION STEPS
1. Import the necessary libraries and the CrossViT model:
   ```python
   import torch
   from vit_pytorch.cross_vit import CrossViT
   ```
2. Initialize the CrossViT model with the desired parameters:
   ```python
   v = CrossViT(
       image_size=256,
       num_classes=1000,
       depth=4,
       sm_dim=192,
       sm_patch_size=16,
       sm_enc_depth=2,
       sm_enc_heads=8,
       sm_enc_mlp_dim=2048,
       lg_dim=384,
       lg_patch_size=64,
       lg_enc_depth=3,
       lg_enc_heads=8,
       lg_enc_mlp_dim=2048,
       cross_attn_depth=2,
       cross_attn_heads=8,
       dropout=0.1,
       emb_dropout=0.1
   )
   ```
3. Create a single-channel image tensor:
   ```python
   img = torch.randn(1, 1, 256, 256)  # Using 1 channel
   ```
4. Attempt to make a prediction using the model:
   ```python
   pred = v(img)  # (1, 1000)
   ```
5. Observe the error message produced during the prediction attempt.
