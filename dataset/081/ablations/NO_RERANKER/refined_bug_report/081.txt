**TITLE**  
CrossViT Fails to Process Non-Three-Channel Images

**SYMPTOMS**  
• The model does not accept images with channel dimensions other than three.  
• Error message:  
  ```bash
  RuntimeError: Given normalized_shape=[768], expected input with shape [*, 768], but got input of size[1, 256, 256]
  ```  
• The model crashes when attempting to process single-channel images, leading to a runtime error during the forward pass.

**EXPECTED BEHAVIOR**  
The CrossViT model should be able to handle images with varying channel dimensions, including single-channel (grayscale) images, without throwing runtime errors.

**REPRODUCTION STEPS**  
1. Import the necessary libraries and the CrossViT model:  
   ```py
   import torch
   from vit_pytorch.cross_vit import CrossViT
   ```
2. Instantiate the CrossViT model with the desired parameters:  
   ```py
   v = CrossViT(
       image_size=256,
       num_classes=1000,
       depth=4,
       sm_dim=192,
       sm_patch_size=16,
       sm_enc_depth=2,
       sm_enc_heads=8,
       sm_enc_mlp_dim=2048,
       lg_dim=384,
       lg_patch_size=64,
       lg_enc_depth=3,
       lg_enc_heads=8,
       lg_enc_mlp_dim=2048,
       cross_attn_depth=2,
       cross_attn_heads=8,
       dropout=0.1,
       emb_dropout=0.1
   )
   ```
3. Create a single-channel image tensor:  
   ```py
   img = torch.randn(1, 1, 256, 256)  # Using 1 channel
   ```
4. Attempt to make a prediction with the model:  
   ```py
   pred = v(img)  # (1, 1000)
   ```  
5. Observe the runtime error that occurs during the forward pass. 

**POSSIBLE SOLUTION**  
To address this issue, consider modifying the model to generalize the input handling for images with varying channel dimensions. Specifically, review and update the code in the following section:  
[Link to the relevant line of code](https://github.com/lucidrains/vit-pytorch/blob/5578ac472faf3903d4739ba783f3875b77177e57/vit_pytorch/cross_vit.py#L178) to allow for dynamic channel input sizes.
