### TITLE
CrossViT Fails to Process Images with Non-Three Channel Formats

### SYMPTOMS
• The model does not accept images with a channel size other than three, leading to a runtime error.  
• The following error message is observed when attempting to process a one-channel image:
```
RuntimeError: Given normalized_shape=[768], expected input with shape [*, 768], but got input of size[1, 256, 256]
```
• The model's behavior is unexpected as it should ideally handle various channel formats.

### EXPECTED BEHAVIOR
The CrossViT model should be able to accept images with any number of channels (e.g., 1, 3, 4) and process them without raising errors. The model should output predictions corresponding to the input image size and channel configuration.

### REPRODUCTION STEPS
1. Import necessary libraries and the CrossViT class:
   ```python
   import torch
   from vit_pytorch.cross_vit import CrossViT
   ```
2. Initialize the CrossViT model with the desired parameters:
   ```python
   v = CrossViT(
       image_size=256,
       num_classes=1000,
       depth=4,
       sm_dim=192,
       sm_patch_size=16,
       sm_enc_depth=2,
       sm_enc_heads=8,
       sm_enc_mlp_dim=2048,
       lg_dim=384,
       lg_patch_size=64,
       lg_enc_depth=3,
       lg_enc_heads=8,
       lg_enc_mlp_dim=2048,
       cross_attn_depth=2,
       cross_attn_heads=8,
       dropout=0.1,
       emb_dropout=0.1
   )
   ```
3. Create a one-channel image tensor:
   ```python
   img = torch.randn(1, 1, 256, 256)  # Using 1 channel
   ```
4. Attempt to get predictions from the model:
   ```python
   pred = v(img)  # Expected output: (1, 1000)
   ```
5. Observe the runtime error that occurs during execution. 

### POSSIBLE SOLUTION
To resolve the issue, the CrossViT implementation may need to be updated to generalize the input handling for images with varying channel counts. The relevant code line that requires modification can be found here:  
[Link to the specific line in the code](https://github.com/lucidrains/vit-pytorch/blob/5578ac472faf3903d4739ba783f3875b77177e57/vit_pytorch/cross_vit.py#L178).
