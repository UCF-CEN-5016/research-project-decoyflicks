TITLE  
TensorFlow 2 Multi-Worker Mirrored Strategy Not Scaling Across Multiple Nodes with GPUs  

SYMPTOMS  
• Throughput does not increase when running ResNet50 training on multi-node multi-GPU cluster using multi_worker_mirrored strategy.  
• Single GPU training throughput is ~270 examples/sec.  
• Multi-GPU single-node (2 GPUs) training scales well to ~700 examples/sec.  
• Multi-node (2 nodes with 2 GPUs each) training throughput remains roughly the same as single GPU (~270 examples/sec), indicating no scaling across nodes.  
• No explicit error messages reported.  
• Logs suggest no improvement in throughput despite correct cluster setup.  

EXPECTED BEHAVIOR  
When using TensorFlow's multi_worker_mirrored strategy across multiple nodes, the training throughput should increase proportionally to the total number of GPUs used across all nodes. For example, running on 2 nodes with 2 GPUs each (4 GPUs total) should ideally show close to a 4x increase in throughput compared to single GPU training, accounting for some communication overhead.  

REPRODUCTION STEPS  
1. Allocate resources on cluster: `salloc --nodes=2 --time=20:00:00 --gres=gpu:2 --partition=gpu --exclusive`  
2. On first node (gpu1), navigate to the model directory and set environment variables:  
   - `cd /home/cdacapp1/scaling/models-master/official/legacy/image_classification`  
   - `export PYTHONPATH="$PYTHONPATH:/home/cdacapp1/scaling/models-master"`  
   - `export DATA_DIR=/scratch/cdacapp1/sowmya/actual_imagenet/tf_records`  
   - `export MODEL_DIR=/home/cdacapp1/scaling/models-master/official/legacy/image_classification/checkpointss`  
3. Activate TensorFlow 2.11 environment:  
   - `source /home/apps/anaconda3/bin/activate`  
   - `conda activate tf11`  
4. Run training on first node with config file `gpu.yaml` specifying:  
   ```
   runtime:
     distribution_strategy: 'multi_worker_mirrored'
     worker_hosts: 'ip1:6655,ip2:6655'
     num_gpus: 2
     task_index: 0
   ```  
5. On second node (gpu2), repeat steps 2 and 3.  
6. Run training on second node with config file `gpu1.yaml` specifying:  
   ```
   runtime:
     distribution_strategy: 'multi_worker_mirrored'
     worker_hosts: 'ip1:6655,ip2:6655'
     num_gpus: 2
     task_index: 1
   ```  
7. Observe throughput remains similar to single GPU run despite using 4 GPUs across 2 nodes.

- TensorFlow GPU: 2.11.0  
- Python: 3.9.7  
- CUDA Driver Version: 450.51.05, CUDA 11.0  
- GPU: Tesla V100-SXM2 16GB
