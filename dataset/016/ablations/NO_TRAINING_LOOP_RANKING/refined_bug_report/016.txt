TITLE  
Multi-Worker Mirrored Strategy Not Scaling Across Nodes in TensorFlow 2.11

SYMPTOMS  
• Throughput does not increase when running ResNet50 on multiple nodes with multi-worker mirrored strategy.  
• Single GPU execution yields approximately 270 examples/sec.  
• Dual GPU execution within a single node achieves approximately 700 examples/sec.  
• Multi-node execution maintains throughput equivalent to a single GPU.  
• No error messages are reported; however, scaling behavior is inconsistent across nodes.

EXPECTED BEHAVIOR  
When using the multi-worker mirrored strategy, the throughput should increase proportionally with the addition of nodes and GPUs, resulting in higher examples processed per second.
