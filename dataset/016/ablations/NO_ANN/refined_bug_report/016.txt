TITLE  
Multi-worker mirrored strategy in TensorFlow 2.11 not scaling across GPUs in different nodes  

SYMPTOMS  
• Throughput does not increase when running the ResNet50 model on multiple nodes with multiple GPUs.  
• Single GPU execution yields approximately 270 examples/sec.  
• Dual GPU execution on a single node achieves approximately 700 examples/sec.  
• Multi-node execution maintains the same throughput as a single GPU, indicating no scaling across nodes.  
• No error messages are reported; however, the expected performance is not achieved.  

EXPECTED BEHAVIOR  
When using the multi-worker mirrored strategy, the throughput should increase proportionally with the addition of GPUs across multiple nodes, resulting in higher examples processed per second compared to a single GPU setup.  

REPRODUCTION STEPS  
1. Allocate resources for two nodes with two GPUs each using the command:  
   `salloc --nodes=2 --time=20:00:00 --gres=gpu:2 --partition=gpu --exclusive`  
2. On the first GPU node (gpu1), navigate to the model directory and set the environment variables:  
   ```
   cd /home/cdacapp1/scaling/models-master/official/legacy/image_classification && \
   export PYTHONPATH="$PYTHONPATH:/home/cdacapp1/scaling/models-master" && \
   export DATA_DIR=/scratch/cdacapp1/sowmya/actual_imagenet/tf_records && \
   export MODEL_DIR=/home/cdacapp1/scaling/models-master/official/legacy/image_classification/checkpointss && \
   source /home/apps/anaconda3/bin/activate && \
   conda activate tf11
   ```  
3. Execute the training script on gpu1 with the appropriate configuration file (gpu.yaml):  
   ```
   python3 classifier_trainer.py --mode=train_and_eval --model_type=resnet --dataset=imagenet --model_dir=$MODEL_DIR --data_dir=$DATA_DIR --config_file=configs/examples/resnet/imagenet/gpu.yaml
   ```  
4. On the second GPU node (gpu2), repeat steps 2 and 3 but use the configuration file (gpu1.yaml) for execution:  
   ```
   python3 classifier_trainer.py --mode=train_and_eval --model_type=resnet --dataset=imagenet --model_dir=$MODEL_DIR --data_dir=$DATA_DIR --config_file=configs/examples/resnet/imagenet/gpu1.yaml
   ```  
5. Monitor the throughput during execution and compare it to single GPU performance.
