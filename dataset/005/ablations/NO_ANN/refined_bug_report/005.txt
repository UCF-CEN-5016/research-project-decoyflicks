• The user encounters an `ImportError` when trying to import 'tensor' from 'tensorflow.python.framework'.

  ImportError: cannot import name 'tensor' from 'tensorflow.python.framework' (D:\project\mobilenet_v4\venv\lib\site-packages\tensorflow\python\framework\__init__.py)
  ```  
• A warning is displayed about the inability to load the dynamic library 'cudart64_110.dll', stating:  
  ```
  Could not load dynamic library 'cudart64_110.dll'; dlerror: cudart64_110.dll not found

• The test fails with the message: "FAILED (errors=1)" after running the mobilenet_test.py file.

The script should successfully run the mobilenet_test.py file without any import errors, allowing the user to dump a checkpoint or h5 file of the MobileNetV4 model.

REPRODUCTION STEPS

2. Install the official TensorFlow models using:  
   ```
   pip3 install tf-models-official
   ```  
3. Copy and run the mobilenet_test.py file.  
4. Observe the error during execution.
