TITLE  
Bug when evaluating images with over 100 objects per image

SYMPTOMS  
• The model fails to converge when training on images containing over 100 objects unless specific configuration lines are included.  
• Adding the line `pipeline_config.eval_input_reader.max_number_of_boxes = 500` via `pipeline_pb2.TrainEvalPipelineConfig()` results in the following error:  
  `AttributeError: 'google.protobuf.pyext._message.RepeatedCompositeContainer' object has no attribute 'max_number_of_boxes'`  
• When the same line is added directly to the `pipeline.config` file, no error occurs, and the model trains normally.  
• Without the specified configuration lines, the model only considers 100 bounding boxes (BB) per image, leading to poor accuracy (e.g., never exceeding 0.04 after 270K training steps) on images with close to 500 objects.

EXPECTED BEHAVIOR  
The model should successfully train and evaluate images with a high number of objects (e.g., 500) without requiring manual edits to the configuration file. It should also converge to a reasonable accuracy based on the provided data.

REPRODUCTION STEPS  
1. Import the necessary TensorFlow and object detection libraries:  
   ```python
   import tensorflow as tf  
   from object_detection.utils import config_util  
   from object_detection.protos import pipeline_pb2  
   from google.protobuf import text_format  
   ```
2. Specify the path to the `pipeline.config` file:  
   ```python
   configpath = #Path to pipeline.config file  
   ```
3. Load the configuration from the `pipeline.config` file:  
   ```python
   config = config_util.get_configs_from_pipeline_file(configpath)  
   ```
4. Create an instance of `TrainEvalPipelineConfig`:  
   ```python
   pipeline_config = pipeline_pb2.TrainEvalPipelineConfig()  
   ```
5. Read the configuration file and merge it into the pipeline configuration:  
   ```python
   with tf.io.gfile.GFile(configpath, "r") as f:  
       proto_str = f.read()  
       text_format.Merge(proto_str, pipeline_config)  
   ```
6. Attempt to set the maximum number of boxes:  
   ```python
   pipeline_config.eval_input_reader.max_number_of_boxes = 500  
   ```  
   (Observe the resulting error message.)
