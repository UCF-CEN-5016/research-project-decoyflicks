TITLE  
Bug when evaluating images with over 100 objects per image

SYMPTOMS  
• The model fails to converge when training on images with more than 100 objects unless specific configuration lines are included.  
• Adding the line `pipeline_config.eval_input_reader.max_number_of_boxes = 500` via `pipeline_pb2.TrainEvalPipelineConfig()` results in the following error:  
  `AttributeError: 'google.protobuf.pyext._message.RepeatedCompositeContainer' object has no attribute 'max_number_of_boxes'`  
• When the line is added directly to the `pipeline.config` file, no error occurs, and the model trains normally.  
• Without the specified configuration lines, the model only detects 100 bounding boxes per image, leading to poor accuracy (e.g., never exceeding 0.04 after 270K training steps) on images containing close to 500 objects.

EXPECTED BEHAVIOR  
The model should successfully converge and produce accurate results when trained on images containing a high number of objects (e.g., 500), provided that the correct configuration settings are applied. The addition of configuration lines via code should not result in errors and should allow for proper model training.

REPRODUCTION STEPS  
1. Import the necessary libraries:  
   ```python
   import tensorflow as tf  
   from object_detection.utils import config_util  
   from object_detection.protos import pipeline_pb2  
   from google.protobuf import text_format  
   ```  
2. Specify the path to the `pipeline.config` file:  
   ```python  
   configpath = #Path to pipeline.config file  
   ```  
3. Load the configurations from the pipeline file:  
   ```python  
   config = config_util.get_configs_from_pipeline_file(configpath)  
   ```  
4. Create an instance of `TrainEvalPipelineConfig`:  
   ```python  
   pipeline_config = pipeline_pb2.TrainEvalPipelineConfig()  
   ```  
5. Read the contents of the `pipeline.config` file:  
   ```python  
   with tf.io.gfile.GFile(configpath, "r") as f:  
       proto_str = f.read()  
       text_format.Merge(proto_str, pipeline_config)  
   ```  
6. Attempt to set `max_number_of_boxes` in the evaluation input reader:  
   ```python  
   pipeline_config.eval_input_reader.max_number_of_boxes = 500  
   ```  
7. Observe the error message that occurs.  
8. Modify the `pipeline.config` file directly to include the necessary lines and retrain the model to confirm successful training.
