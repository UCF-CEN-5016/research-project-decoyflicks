TITLE  
Bug in Object Detection Model Configuration for Images with Over 100 Objects

SYMPTOMS  
• The model fails to converge when training on images containing more than 100 objects unless specific configuration lines are included.  
• Error message encountered:  
  `AttributeError: 'google.protobuf.pyext._message.RepeatedCompositeContainer' object has no attribute 'max_number_of_boxes'`  
• Without the specified configuration lines, the model only processes 100 bounding boxes per image, resulting in poor performance (accuracy never exceeding 0.04).  
• The issue worsens with increased object clustering in images.

EXPECTED BEHAVIOR  
The model should successfully train and converge on images with more than 100 objects when the appropriate configuration settings are applied. It should allow for the processing of up to 500 bounding boxes without throwing errors or limiting detection capabilities.

REPRODUCTION STEPS  
1. Import the necessary libraries:  
   ```python
   import tensorflow as tf  
   from object_detection.utils import config_util  
   from object_detection.protos import pipeline_pb2  
   from google.protobuf import text_format  
   ```
2. Set the path to the `pipeline.config` file.  
   ```python
   configpath = #Path to pipeline.config file  
   ```
3. Load the configuration from the pipeline file:  
   ```python
   config = config_util.get_configs_from_pipeline_file(configpath)  
   ```
4. Initialize the `TrainEvalPipelineConfig` object:  
   ```python
   pipeline_config = pipeline_pb2.TrainEvalPipelineConfig()  
   ```
5. Read the contents of the `pipeline.config` file:  
   ```python
   with tf.io.gfile.GFile(configpath, "r") as f:  
       proto_str = f.read()  
       text_format.Merge(proto_str, pipeline_config)  
   ```
6. Attempt to set the `max_number_of_boxes` parameter:  
   ```python
   pipeline_config.eval_input_reader.max_number_of_boxes = 500  
   ```
7. Observe the error message that arises from this operation.  
8. Directly modify the `pipeline.config` file to include the necessary configuration lines and verify that the model trains successfully without errors.
