TITLE  
MMS Forced Alignment: log_probs Shape Mismatch Error

SYMPTOMS  
• Encountered a RuntimeError while attempting to perform forced alignment on an audio file with its text transcription.  
• Error message displayed: `RuntimeError: log_probs must be 3-D (batch_size, input length, num classes)`.  
• The issue appears to have originated after a specific commit in the repository.

EXPECTED BEHAVIOR  
The forced alignment process should successfully align the audio file with the provided text transcription without generating any shape-related errors, resulting in the output being saved in the specified output directory.

REPRODUCTION STEPS  
1. Run the command: `python align_and_segment.py --audio_filepath audio.wav --text_filepath text.txt --lang udm --outdir output --uroman uroman/bin`.  
2. Ensure that the audio file (`audio.wav`) and text file (`text.txt`) are correctly formatted and accessible.  
3. Observe the output for any error messages related to log_probs shape.
