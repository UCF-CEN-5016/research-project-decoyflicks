**TITLE**  
Dimension mismatch error when predicting with converted NER model in TFLite format

**SYMPTOMS**  
• The model conversion from Keras to TFLite is successful, but prediction fails.  
• Error message encountered:  
  ```
  ValueError: Cannot set tensor: Dimension mismatch. Got 9 but expected 1 for dimension 1 of input 0.
  ```  
• The input dimension of the TFLite model is fixed to **int64[1,1]**, which does not match the expected input shape for predictions.

**EXPECTED BEHAVIOR**  
The model should accept input of the correct shape and return predictions without any dimension mismatch errors.

**REPRODUCTION STEPS**  
1. Train the NER model following the documentation provided in the Keras example.  
2. Convert the trained model to TFLite format using the provided conversion code.  
3. Attempt to make a prediction using a sample input that has a different shape than expected by the model.  
4. Observe the dimension mismatch error during the prediction step.
