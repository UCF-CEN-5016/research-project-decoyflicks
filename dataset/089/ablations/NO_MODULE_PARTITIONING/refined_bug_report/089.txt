TITLE  
Conflict between `qk_norm` and `kv_heads` settings in x-transformers

SYMPTOMS  
• When using `qk_norm` alongside `kv_heads` not equal to `heads`, the model exhibits unexpected behavior.  
• No specific error messages were reported, but there is a noticeable malfunction in the attention mechanism.  
• The output does not align with expected results when both parameters are applied simultaneously.

EXPECTED BEHAVIOR  
The model should function correctly with `qk_norm` and allow for `kv_heads` to differ from `heads` without any conflicts, producing accurate attention outputs.

REPRODUCTION STEPS  
1. Set `qk_norm` to `True` in the model configuration.  
2. Set `kv_heads` to a value different from `heads`.  
3. Run the model to observe the output and any discrepancies.  
4. Compare the output with the expected behavior when `kv_heads` equals `heads`.  
5. Verify the model's performance and output consistency across various input samples.
