TITLE  
Conflict between `qk_norm` and `kv_heads` when `kv_heads != heads`

SYMPTOMS  
• The implementation of `qk_norm` does not function correctly when the number of key-value heads (`kv_heads`) is different from the number of heads (`heads`).  
• The model may produce incorrect outputs or fail to initialize properly.  
• No specific error messages were reported, but unexpected behavior was noted in the model's performance.

EXPECTED BEHAVIOR  
When both `qk_norm` and `kv_heads` are used, the model should correctly normalize the query-key interactions without any conflicts, allowing for flexible configurations of heads.

REPRODUCTION STEPS  
1. Set up the model with `kv_heads` set to a value different from `heads`.  
2. Enable the `qk_norm` feature in the model configuration.  
3. Run a forward pass with sample input data.  
4. Observe the model's output and any initialization errors or unexpected behaviors.
