TITLE  
Retina Net Model Fit Error with Custom Dataset  

SYMPTOMS  
• An error occurs when attempting to fit the Retina Net model with a custom dataset structured as per the Keras documentation.  
• The following error message is displayed:  
```
InvalidArgumentError: Graph execution error: Detected at node retina_net_label_encoder_18/GatherV2_1 [...]
```
• The error indicates an issue with the indices used in the model training process, specifically:  
```
indices[3,63949] = 0 is not in [0, 0)
```

EXPECTED BEHAVIOR  
The model should successfully fit to the provided dataset without errors, allowing for training to proceed as intended, and producing valid output metrics.

REPRODUCTION STEPS  
1. Structure your dataset according to the guidelines provided in the Keras documentation at https://keras.io/guides/keras_cv/object_detection_keras_cv/.  
2. Load the dataset into the training and evaluation datasets (`train_ds` and `eval_ds`).  
3. Execute the following code to fit the model:  
```python
model.fit(
    train_ds.take(20),
    validation_data=eval_ds.take(20),
    epochs=1,
    callbacks=[EvaluateCOCOMetricsCallback(eval_ds.take(20))],
)
```  
4. Observe the output for any errors during the fitting process.
