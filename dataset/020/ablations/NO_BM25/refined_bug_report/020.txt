TITLE  
Compatibility issue between tf-models-official==2.12.0 and Python 3.8

SYMPTOMS  
• Encountered a `TypeError` during training and evaluation in the Object Detection tutorial.  
• Error message:  
  ```
  TypeError: unsupported operand type(s) for |: 'dict' and 'dict'
  ```  
• The error occurs specifically in the `eval_step` function of `base_trainer.py` when attempting to merge two dictionaries using the `|` operator.

EXPECTED BEHAVIOR  
When using `tf-models-official==2.12.0` with Python 3.8, the training and evaluation process should execute without errors, successfully returning the model and evaluation logs.

REPRODUCTION STEPS  
1. Follow the Object Detection with Model Garden tutorial found at [TensorFlow Object Detection Tutorial](https://www.tensorflow.org/tfmodels/vision/object_detection).  
2. Set up the environment using the Docker image `tensorflow/tensorflow:2.12.0-gpu-jupyter`.  
3. Execute the following code snippet:  
   ```python
   model, eval_logs = tfm.core.train_lib.run_experiment(
       distribution_strategy=distribution_strategy,
       task=task,
       mode='train_and_eval',
       params=exp_config,
       model_dir=model_dir,
       run_post_eval=True)
   ```  
4. Observe the error message regarding the unsupported operand type(s) for the `|` operator.  

ADDITIONAL CONTEXT  
It appears that the issue may be related to the compatibility between `tf-models-official==2.12.0` and Python 3.8, particularly in how dictionary merging is handled in this version of TensorFlow. For further reference, a similar issue can be found here: [Stack Overflow discussion on TypeError](https://stackoverflow.com/questions/62498581/typeerror-when-merging-dictionaries-unsupported-operand-types-for-dict-a).
