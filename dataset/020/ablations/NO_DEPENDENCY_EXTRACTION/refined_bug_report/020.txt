**TITLE**  
Compatibility issue between tf-models-official==2.12.0 and Python 3.8 causing TypeError during training and evaluation.

**SYMPTOMS**  
• Encountered a `TypeError` during the training and evaluation phase of the Object Detection tutorial.  
• Error message:  
  ```
  TypeError: in user code:
      File "/usr/local/lib/python3.8/dist-packages/official/core/base_trainer.py", line 442, in eval_step  *
          return passthrough_logs | logs
      TypeError: unsupported operand type(s) for |: 'dict' and 'dict'
  ```  
• The issue arises when executing the `run_experiment` function from the training library.

**EXPECTED BEHAVIOR**  
The training and evaluation process should complete without errors, and the model should train and evaluate as described in the tutorial.

**REPRODUCTION STEPS**  
1. Follow the Object Detection with Model Garden tutorial at https://www.tensorflow.org/tfmodels/vision/object_detection.  
2. Set up the environment using the Docker image `tensorflow/tensorflow:2.12.0-gpu-jupyter`.  
3. Execute the following code:  
   ```python
   model, eval_logs = tfm.core.train_lib.run_experiment(
       distribution_strategy=distribution_strategy,
       task=task,
       mode='train_and_eval',
       params=exp_config,
       model_dir=model_dir,
       run_post_eval=True)
   ```  
4. Observe the error message indicating a `TypeError` related to the use of the `|` operator on dictionaries.
