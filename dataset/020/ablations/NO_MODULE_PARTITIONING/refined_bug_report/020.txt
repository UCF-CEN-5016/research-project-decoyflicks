TITLE  
Conflict between tf-models-official==2.12.0 and Python 3.8 resulting in TypeError during training evaluation.

SYMPTOMS  
• Encountered a TypeError during the training and evaluation phase of the Object Detection tutorial.  
• Error message:  
  ```
  TypeError: in user code:
      File "/usr/local/lib/python3.8/dist-packages/official/core/base_trainer.py", line 442, in eval_step  *
          return passthrough_logs | logs
      TypeError: unsupported operand type(s) for |: 'dict' and 'dict'
  ```  
• The issue arises when executing the `tfm.core.train_lib.run_experiment` function.

EXPECTED BEHAVIOR  
The training and evaluation process should execute without errors, successfully returning training and evaluation logs.

REPRODUCTION STEPS  
1. Follow the Object Detection with Model Garden tutorial at https://www.tensorflow.org/tfmodels/vision/object_detection.  
2. Execute the following code snippet:  
   ```python
   model, eval_logs = tfm.core.train_lib.run_experiment(
       distribution_strategy=distribution_strategy,
       task=task,
       mode='train_and_eval',
       params=exp_config,
       model_dir=model_dir,
       run_post_eval=True)
   ```  
3. Observe the error message indicating the TypeError related to unsupported operand types.  

ADDITIONAL CONTEXT  
• The issue appears to be related to compatibility between `tf-models-official==2.12.0` and Python 3.8.  
• Reference link for similar issues: [Stack Overflow TypeError](https://stackoverflow.com/questions/62498581/typeerror-when-merging-dictionaries-unsupported-operand-types-for-dict-a)  
• Environment is based on the Docker image: `tensorflow/tensorflow:2.12.0-gpu-jupyter`.
